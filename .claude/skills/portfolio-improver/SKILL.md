# Portfolio Improver

## Purpose

Improve Aphisit Hathaway's personal digital marketing portfolio while keeping it minimal, professional, recruiter-friendly, and suitable for GitHub Pages.

This portfolio represents Aphisit as a digital marketing graduate with hands-on entrepreneurial experience, SEO knowledge, content strategy skills, analytics awareness, and practical AI-assisted workflow capability.

## Repository Context

Before editing, read:

- `CLAUDE.md`
- `index.html`
- `README.md`

The current website is intentionally simple and self-contained. Do not add frameworks or build tools unless explicitly requested.

## Core Rules

1. Never push directly to `main`.
2. Always create a new branch and open a Pull Request for code changes.
3. Keep the site static and compatible with GitHub Pages.
4. Preserve the single-page structure unless the user explicitly asks for a larger refactor.
5. Do not add React, Next.js, Vue, Tailwind, npm packages, or build tools unless explicitly requested.
6. Do not edit secrets, tokens, GitHub Actions secrets, or sensitive configuration.
7. Do not remove existing personal positioning unless replacing it with stronger recruiter-focused copy.
8. Keep changes focused and easy to review.

## Design Direction

The design should be:

- Minimal
- Clean
- Modern
- Professional
- Mobile-responsive
- Easy to scan
- Suitable for recruiters and hiring managers

Avoid:

- Overly colorful visuals
- Heavy animations
- Generic template language
- Complex layouts
- Unnecessary sections
- Claims that sound exaggerated or unsupported

## Content Direction

Position Aphisit as:

A digital marketing graduate with practical experience in SEO, content strategy, performance marketing, analytics, event coordination, entrepreneurial projects, and AI-assisted workflows.

Copywriting should be:

- Direct
- Specific
- Recruiter-friendly
- Outcome-focused
- Professional
- Not exaggerated

Prefer concrete phrases such as:

- SEO content planning
- campaign performance tracking
- portfolio-first learning
- AI-assisted workflow
- practical marketing execution
- data-informed content decisions

Avoid vague phrases such as:

- passionate marketer
- highly motivated individual
- marketing guru
- growth hacker
- world-class expert

## Improvement Process

When asked to improve the portfolio:

1. Read the existing `index.html`.
2. Identify the current sections and content.
3. Propose a short plan before making major changes.
4. Make focused edits only.
5. Preserve working HTML/CSS/JS.
6. Check mobile responsiveness.
7. Improve SEO basics where appropriate.
8. Open a Pull Request with a clear summary.

## SEO Checklist

When improving SEO, check:

- Page title
- Meta description
- Open Graph title
- Open Graph description
- Heading structure
- Clear hero message
- Descriptive section headings
- Recruiter-relevant keywords
- No keyword stuffing

## Pull Request Format

When opening a Pull Request, include:

## Summary

Briefly explain what changed.

## Why

Explain why the change improves the portfolio.

## Changes Made

- List key changes
- Mention edited files

## Safety Check

- No secrets edited
- No frameworks added
- GitHub Pages compatibility preserved
- Main branch not modified directly
